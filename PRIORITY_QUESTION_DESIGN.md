# 优先问题设计改进

## 改进目标
确保所有图表公式类型的section都有一个优先问题，回答"是什么"和"原理是什么"。

## 核心要求

### 对于 type="figure" 或 "equation" 的section

`sub_questions` 数组的**第一个问题**必须同时回答：
1. **这是什么**（内容描述）
2. **原理是什么**（工作机制/计算逻辑）

### 问题结构

```json
{
  "sub_questions": [
    {
      "question": "【优先】这个图/表/公式展示了什么？...其工作原理/计算原理是什么？",
      "question_type": "phenomenon"
    },
    {
      // 其他维度问题（mechanism, critique等）
    }
  ]
}
```

## 示例

### 图表优先问题
```
【优先】Fig 1展示了什么内容？图中各个元素（形状、目标、时序）代表什么含义？这个任务的工作原理/流程是怎样的？
```

### 公式优先问题
```
【优先】Equation 1的数学表达式是什么？各个符号代表什么含义？这个公式的计算原理和推导逻辑是怎样的？
```

## 问题顺序

1. **第一位**：优先问题（是什么 + 原理）
2. **第二位起**：其他维度问题
   - mechanism（机理深究）
   - critique（批判改进）
   - 等等

## 自检清单

生成完sub_questions后，检查：
- [ ] 第一个问题是否包含"是什么"？
- [ ] 第一个问题是否包含"原理"？
- [ ] question_type 是否为 "phenomenon"？
- [ ] 是否还有其他维度的问题？

## 预期效果

- ✅ 每个图表分析都从基础描述开始
- ✅ 确保读者先理解"是什么"再深入分析
- ✅ 原理解释作为强制问题，不会遗漏
- ✅ 其他维度问题作为深化补充
